# Screenshots

Drop captures here using the filenames referenced from the top-level README:

| Filename | What to capture | Recommended size |
|---|---|---|
| `profile-desktop.png` | Public profile page, desktop viewport, above the fold | 1440 × 900 |
| `profile-mobile.png` | Public profile page, iPhone-sized viewport (390 × 844) | 390 × 844 |
| `admin-dashboard.png` | `/admin` employee list with at least 3–5 seed employees | 1440 × 900 |
| `analytics-detail.png` | Per-employee analytics drill-down (`/admin/analytics/<slug>`) | 1440 × 900 |
| `qr-share.png` | Public profile with the QR / share modal open | 1440 × 900 (or 390 × 844) |
| `employee-portal.png` | Employee self-service editor | 1440 × 900 |

## Tips

- Use a clean, demo-friendly dataset. The seed script creates one admin; add a few employees with realistic but fictional names + photos before capturing.
- Crop tightly. The README renders these in a 3-up grid — extreme widths look uneven.
- PNG is fine for static screens. For a short walkthrough, commit a sub-5 MB `demo.gif` and reference it from the README, or upload to YouTube / Loom and embed the thumbnail link.
- Strip any personal data from the captures before committing.
